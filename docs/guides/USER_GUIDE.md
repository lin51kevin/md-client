## 1. 入门指南

### 1.1 安装与启动

**安装 MarkLite**：
- 从 [GitHub Releases](https://github.com/lin51kevin/md-client/releases) 下载对应平台的安装包
- macOS：双击 `.dmg` 文件，拖拽 MarkLite 到 Applications 文件夹
- Windows：运行 `.exe` 安装程序，按提示完成安装
- Linux：下载 `.AppImage` 文件，赋予执行权限：`chmod +x MarkLite-*.AppImage`

**从命令行启动**：
```bash
# macOS
open -a MarkLite /path/to/file.md

# Windows
MarkLite.exe C:\path\to\file.md

# Linux
./MarkLite-*.AppImage /path/to/file.md
```

### 1.2 第一次使用

1. **打开一个 Markdown 文件**或创建新文件（`Ctrl/Cmd + N`）
2. **熟悉三种视图**：
   - 📝 **编辑** — 仅显示编辑器
   - 📖 **分栏**（默认）— 编辑器与预览并排，实时同步
   - 👁️ **预览** — 仅显示渲染结果
3. **尝试编辑**：输入 Markdown，在预览区看到实时效果
4. **自动保存**：编辑 1 秒无变化后自动保存到磁盘

### 1.3 打开文件夹

MarkLite 支持**文件夹工作区**模式：

1. 点击 **Toolbar** 左上的**文件夹图标**，或按 `Ctrl+O`
2. 选择包含 Markdown 文件的文件夹
3. **文件树侧边栏**出现在左侧，显示所有 `.md` / `.markdown` / `.txt` 文件
4. 在文件树中**双击**任意文件快速打开
5. 使用**上级目录**按钮返回上级文件夹，或**刷新**按钮重载文件树

## 2. 编辑器基础

### 2.1 编辑操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 新建文件 | `Ctrl+N` | 创建无标题新文件 |
| 打开文件 | `Ctrl+O` | 打开单个 `.md` 文件 |
| 打开文件夹 | `Ctrl+Shift+O` | 打开文件夹（显示文件树） |
| 保存文件 | `Ctrl+S` | 手动保存（自动保存已启用） |
| 另存为 | `Ctrl+Shift+S` | 保存为新文件 |
| 关闭标签页 | `Ctrl+W` | 关闭当前编辑标签页 |
| 下一个标签页 | `Ctrl+Tab` | 导航到下一个标签页 |
| 上一个标签页 | `Ctrl+Shift+Tab` | 导航到上一个标签页 |

### 2.2 视图模式

**三种编辑视图** — 点击 Toolbar 右侧的视图按钮或使用快捷键切换：

| 模式 | 快捷键 | 说明 |
|------|--------|------|
| 📝 编辑 | `Ctrl+Alt+1` | 仅显示编辑器，最大化空间 |
| 📖 分栏 | `Ctrl+Alt+2` | 编辑器 + 预览并排，实时同步滚动 |
| 👁️ 预览 | `Ctrl+Alt+3` | 仅显示预览区，只读视图 |

### 2.3 焦点模式

在高度专注的环境下写作，隐藏 UI 干扰。Toolbar 右侧点击或快捷键激活：

| 模式 | 快捷键 | 说明 |
|------|--------|------|
| 📝 打字机模式 | `Ctrl+Shift+M` | 当前行始终居中，隐藏工具栏和状态栏 |
| 🎯 专注模式 | `Ctrl+Shift+F` | 仅显示编辑器，暗化周围 UI |
| 🔲 全屏模式 | `F11` | 占满整个屏幕（ESC 退出） |

### 2.4 主题切换

点击 Toolbar 右侧的**太阳/月亮图标**，或按 `Ctrl+Shift+T` 切换主题：

- ☀️ **亮色** — GitHub Light 风格
- 🌙 **暗色** — GitHub Dark 风格
- 📜 **Sepia** — 护眼棕色调
- 🔲 **高对比度** — High Contrast 模式
- 🖥️ **跟随系统**（默认）— 自动跟随操作系统深色模式设置

## 3. 功能详解

### 3.1 编辑工具栏

**Toolbar** 提供一键格式化和快捷操作：

#### 文本格式（Text Formatting）
| 按钮 | 快捷键 | 效果 |
|------|--------|------|
| **B** 加粗 | `Ctrl+B` | 将选中文本包装为 `**bold**` |
| *I* 斜体 | `Ctrl+I` | 将选中文本包装为 `*italic*` |
| ~~S~~ 删除线 | `Ctrl+Shift+X` | 将选中文本包装为 `~~strikethrough~~` |
| `<>` 代码 | `Ctrl+E` | 将选中文本包装为 `` `code` `` |

#### 块级元素（Block Elements）
| 按钮 | 效果 |
|------|------|
| # 标题 | 循环升级标题级别（H1→H2→…→H6→无前缀），支持多行选区 |
| > 引用 | 在行前添加 `> ` 标记为引用块 |
| • 无序列表 | 在行前添加 `- ` 创建列表项，支持多行选区 |
| 1. 有序列表 | 在行前添加 `1. ` 创建编号列表，支持多行选区 |
| ☑ 任务列表 | 在行前添加 `- [ ] ` 创建任务项，再次点击移除前缀 |
| `{}` 代码块 | 有选区时包裹为围栏代码块，无选区时插入空代码块 |
| — 分割线 | 在光标所在行后插入 `---` 水平分割线 |
| Σ 数学公式 | 有选区时插入行内公式 `$...$`，无选区时插入块级公式 `$$...$$` |

#### 插入元素（Insertion）
| 按钮 | 效果 |
|------|------|
| 🔗 链接 | 弹出对话框，输入链接文本和 URL，插入 `[text](url)` |
| 🖼️ 图片（本地） | 打开文件选择器，选择图片文件插入 `![alt](path)` 链接 |
| 🖼️ 图片（链接） | 弹出对话框，输入图片 URL，插入 `![alt](url)` |
| 📊 表格 | 弹出 **尺寸选择器**（见下文）快速插入 Markdown 表格 |

#### 表格尺寸选择器

点击工具栏中的表格按钮（📊）弹出网格选择器：

1. 用鼠标悬停在网格上，高亮区域实时显示将生成的行列数（如 `3×4`）
2. 单击目标大小，编辑器立即插入带表头的完整表格：

```markdown
| Col 1 | Col 2 | Col 3 |
| --- | --- | --- |
|  |  |  |
|  |  |  |
|  |  |  |
```

3. 网格最大支持 8×8；初始显示 5×5，悬停接近边缘时自动扩展
4. 点击选择器外部任意位置关闭弹窗

### 3.2 右键上下文菜单

在编辑器内**右键单击**，出现上下文感知的菜单，包含 20+ 操作：

**通用操作**：
- ✂️ 剪切 / 📋 复制 / 📌 粘贴 / 🎯 全选
- 撤销 / 重做

**上下文操作**（根据光标位置自动显示相关操作）：

**在标题行**：
- ⬆️ 提升标题（H2 → H1）
- ⬇️ 降低标题（H1 → H2）
- 转换为其他标题级别

**在列表内**：
- ↪️ 缩进列表项
- ↩️ 反缩进列表项
- 转换列表类型（有序 ↔ 无序）

**在表格内**：
- ➕ 上方/下方插入行
- ➕ 左方/右方插入列
- ❌ 删除行/列
- ⬅️ 左对齐 / ⬆️ 中对齐 / ➡️ 右对齐

**在代码块/数学公式**：
- 📋 复制代码块内容

### 3.3 活动栏与侧边栏（ActivityBar）

MarkLite 使用 **VS Code 风格的活动栏**（窗口最左侧的图标列），统一管理所有侧边栏面板：

| 图标 | 面板 | 说明 |
|------|------|------|
| 📁 | 文件树 | 显示打开文件夹中的所有文件 |
| 🔍 | 全局搜索 | 跨文件搜索与替换 |
| 📋 | 大纲（TOC） | 当前文档标题结构 |
| 🌿 | Git | Git 状态与提交操作 |
| 🧩 | 插件 | 插件管理与安装 |

**操作方式**：
- **单击**活动栏图标：切换显示对应侧边栏面板（再次单击同一图标可折叠）
- 插件可在活动栏注册自定义面板图标

### 3.4 文件树侧边栏

打开文件夹后，点击活动栏的**文件夹图标**，左侧显示**文件树**：

**文件树功能**：
- 📁 **文件夹展开/折叠** — 点击文件夹名左侧的 ▶️ 箭头
- 📄 **双击打开文件** — 在编辑器中打开选中的 `.md` / `.txt` 文件
- 🔍 **文件悬浮预览** — 鼠标悬停文件名时，弹出浮窗预览该文件前 10 行内容
- 🔄 **刷新** — 刷新按钮重载文件树（新增文件夹时使用）
- 📂 **选择其他文件夹** — 使用上级目录或选择按钮切换工作区
- 🔍 **文件筛选** — 仅显示 `.md`、`.markdown`、`.txt` 文件（自动过滤其他文件类型）

### 3.5 跨文件搜索

在打开文件夹的状态下，点击活动栏的**搜索图标**（🔍）或按 `Ctrl+Shift+F`：

1. **搜索面板**出现在右侧
2. 输入搜索关键词
3. 勾选 **Aa**（大小写敏感）或 **.*（正则表达式）**
4. 显示匹配结果（最多 200 条），结果格式为 `文件名 : 行号 : 匹配行内容`
5. **点击任意结果**快速跳转到该文件和行号

### 3.6 大纲侧边栏（TOC）

点击活动栏的**大纲图标**，显示当前文档的**目录大纲**，自动提取所有标题（H1-H6）：

**大纲功能**：
- 📍 **点击跳转** — 点击任意标题快速跳转到文档中该位置
- 🎚️ **折叠/展开** — 点击标题前的 ▶️ 箭头折叠其子标题
- 🔄 **实时更新** — 编辑时自动更新大纲结构

### 3.7 表格可视化编辑

**双击任意表格**（在编辑模式）或在预览区**点击表格编辑按钮**，弹出编辑对话框：

**表格编辑功能**：
- ➕ **添加行** — 在表格上方或下方插入新行
- ➖ **删除行** — 删除指定行
- ➕ **添加列** — 在表格左方或右方插入新列
- ➖ **删除列** — 删除指定列
- ⬅️ / ⬆️ / ➡️ **对齐方式** — 改变列的文本对齐（左/中/右）
- ✏️ **编辑单元格** — 点击单元格直接编辑内容

对话框关闭后，修改内容自动更新到编辑器中的 Markdown 表格。

### 3.7 图片粘贴与拖拽

**粘贴图片**：
1. 将图片复制到剪贴板（或从文件管理器拖拽）
2. 在编辑器中光标定位到要插入图片的位置
3. **粘贴**（`Ctrl+V`）或**拖拽**图片文件到编辑器
4. 图片自动保存到 `assets/images/` 文件夹，生成 Markdown 格式 `![alt](path)`

**支持的图片格式**：PNG、JPG、JPEG、GIF、WebP

### 3.8 搜索与替换

按 `Ctrl+F` 打开**查找工具栏**：

- 输入搜索词，编辑器中自动高亮所有匹配项
- 勾选 **Aa** 开启大小写敏感搜索
- 勾选 **.*\*** 启用正则表达式搜索
- ↑ / ↓ 按钮导航到前一个/后一个匹配项

按 `Ctrl+H` 打开**替换工具栏**：

- 在"查找"字段输入搜索词，"替换"字段输入替换内容
- 🎯 **替换** — 替换当前匹配项
- 🔄 **全部** — 一次性替换所有匹配项

### 3.9 版本快照（本地历史）

**自动快照**：
- 每次按 `Ctrl+S` 手动保存时，自动创建当前文档的快照
- 最多保留最近 20 个快照

**访问快照**：
1. 点击状态栏右侧的**版本数字**（如 `5 snapshots`）
2. 弹出快照列表，显示创建时间戳
3. 点击任意快照预览其内容
4. 如要恢复，点击**恢复**按钮

### 3.10 写作统计

**实时统计面板**显示在状态栏右侧，包含：

| 指标 | 说明 |
|------|------|
| 字数（Words） | 文档中单词/中文字符总数 |
| 字符数（Characters） | 包含空格的总字符数 |
| 句子数（Sentences） | 按 `.!?;:` 分割的句子数 |
| 阅读时间（Reading Time） | 按 350 字/分钟计算的预计阅读时间 |

### 3.11 命令面板

**快速命令执行**，按 `Ctrl+Shift+P` 打开命令面板：

**功能特性**：
- 🔍 **模糊搜索** — 输入关键词快速查找命令（支持部分匹配）
- ⭐ **最近命令优先** — 常用命令排序靠前，提升效率
- 📋 **20+ 内置命令** — 包括：
  - 文件操作：新建、打开、保存、导出
  - 编辑操作：撤销、重做、全选、查找
  - 视图控制：切换视图模式、焦点模式
  - 主题/语言：切换主题、切换语言
  - 快捷键查看

**使用示例**：
1. 按 `Ctrl+Shift+P` 打开面板
2. 输入 `export` 查找所有导出相关命令
3. 上下箭头选择，按 Enter 执行
4. 或直接模糊匹配：输入 `pdf` 快速定位 PDF 导出

### 3.12 Wiki-Link 导航

**文档间快速导航**，使用 `[[target]]` 语法：

**语法与功能**：
- **基本语法** — 在 Markdown 中输入 `[[filename]]` 创建 wiki 链接
  - 示例：`[[项目计划]]` 链接到同文件夹中的 `项目计划.md`
  - 自动识别已存在的文档（不需要 `.md` 扩展名）

- **点击导航**：
  - **单击** — 打开链接指向的文档
  - **Ctrl+Click** — 在新标签页打开文档

- **自动创建**：
  - 如果链接指向的文档不存在，Ctrl+Click 会提示创建新文档
  - 确认后自动创建并打开新文件

- **链接高亮**：
  - wiki 链接在编辑器中以特殊颜色高亮显示，便于识别

**应用场景**：
- 个人知识库 — 使用 wiki 链接互连各主题笔记
- 项目文档 — 快速导航相关的需求、设计、代码笔记
- 学习笔记 — 关联相关概念和章节

### 3.13 片段管理器

**文本模板与动态变量替换**，快速插入常用内容：

**打开片段管理器**：
- 点击 Toolbar 中的**片段图标**（或通过命令面板搜索 `Snippets`）
- 右侧弹出片段列表面板

**创建片段**：
1. 点击 **➕ 新建片段** 按钮
2. 输入片段名称（如 `签名`、`代码块`）
3. 输入片段内容，支持以下动态变量：
   - `${date}` — 当前日期（YYYY-MM-DD 格式）
   - `${time}` — 当前时间（HH:mm 格式）
   - `${filename}` — 当前文件名（不含路径）
   - `${cursor}` — 片段插入后光标位置（执行后光标停留处）
4. 保存片段

**使用片段**：
1. 在编辑器中定位光标到要插入位置
2. 双击片段列表中的片段名，或点击片段后的插入按钮
3. 片段内容自动替换动态变量后插入
4. 光标自动定位到 `${cursor}` 标记的位置

**管理片段**：
- **编辑** — 点击片段名旁的编辑图标修改内容
- **删除** — 点击片段名旁的删除图标移除片段
- **本地存储** — 所有片段保存到浏览器 localStorage，不同文件夹自动同步

**片段示例**：

| 片段名 | 内容 | 用途 |
|--------|------|------|
| `签名` | `——${date} 于 ${filename}` | 快速添加文档签名 |
| `代码块` | ` ```\n${cursor}\n``` ` | 快速插入代码块 |
| `待办` | `- [ ] ${cursor} (${date})` | 创建待办事项 |
| `邮件` | `您好，\n\n${cursor}\n\n祝好！` | 快速起草邮件 |

### 3.14 多光标编辑

**在多个位置同时编辑**，大幅提升编辑效率：

**快捷键**：

| 快捷键 | 功能 |
|--------|------|
| `Alt+D` | 选中当前单词的所有出现 |
| `Alt+Up` | 在上一行添加光标 |
| `Alt+Down` | 在下一行添加光标 |

**使用示例**：

1. **批量替换单词**：
   - 光标定位到某单词（如 `variable`）
   - 按 `Alt+D`，自动选中文件中所有 `variable` 出现
   - 开始输入新名称，所有位置同时替换

2. **竖列编辑**：
   - 点击第一行末尾定位光标
   - 按 `Alt+Down` 在下一行添加光标
   - 继续按 `Alt+Down` 在更多行添加光标
   - 输入或删除内容，所有位置同时变化

3. **列表缩进**：
   - 选中多行列表项（或用 `Alt+Down` 为每行添加光标）

### 3.21 Milkdown WYSIWYG 预览

> **v0.8.0 新功能**

MarkLite v0.8.0 将预览引擎升级为 **Milkdown Crepe**，分栏模式下预览区支持**所见即所得（WYSIWYG）直接编辑**。

**核心特性**：
- ✏️ **预览区直接编辑** — 无需切换到编辑器，在预览区点击即可原地修改文本
- 🔄 **双向同步** — 预览区的修改实时同步到左侧 CodeMirror 编辑器，编辑器的修改也实时更新预览
- 🎛️ **内置 Toolbar** — 选中文本后弹出格式化工具栏（加粗、斜体、行内代码、链接等）

**AST 精准块级编辑（Preview-Edit 插件）**：
- 所有块级元素（段落、标题、引用块、列表项、代码块）均可点击进入编辑模式

- 支持完整的**撤销/重做**（`Ctrl+Z` / `Ctrl+Y`）
- 按 `Enter` 确认编辑，按 `Esc` 取消

**如何使用**：
1. 切换到**分栏视图**（`Ctrl+Alt+2` 或 `Ctrl+2`）
2. 在预览区点击任意文本段落 — 该段落变为可编辑的 textarea
3. 修改内容后按 `Enter` 或点击其他区域确认
4. 或选中文本后使用弹出的 **InlineToolbar** 应用格式

## 5. 常见问题

### Q1: 如何启用 Vim 模式？

A: 点击 Toolbar 右侧的 **Terminal 图标** 按钮即可一键开启/关闭 Vim 模式。开启后，编辑器使用 Vim 键盘映射，支持 `hjkl` 导航、`dd` 删除行等标准 Vim 操作。

### Q2: 图片粘贴后保存到哪里？

A: 图片自动保存到当前文件夹的 `assets/images/` 子目录。如果该目录不存在会自动创建。

### Q3: 文件分栏预览的滚动不同步？

A: 这是正常现象。MarkLite 会尽力同步编辑区和预览区的滚动位置，但由于渲染高度不同可能出现微小偏差。如需精确定位，可使用**大纲侧边栏**中的标题链接快速跳转。

### Q4: 能否使用 Jupyter Notebook 或其他格式？

A: MarkLite 目前仅支持 Markdown（`.md` / `.markdown`）和纯文本（`.txt`）格式。

### Q5: 支持哪些 Markdown 扩展？

A: MarkLite 支持以下扩展：
- **GFM**（GitHub Flavored Markdown）— 表格、删除线、任务列表等
- **KaTeX** — 数学公式（`$...$` 行内，`$$...$$` 块级）
- **Mermaid** — 流程图、时序图、状态图等
- **脚注** — `[^1]` 脚注定义和引用
- **Frontmatter** — YAML 元数据块
- **自定义容器** — `::: info` 等指令块
- **代码高亮** — highlight.js 支持 100+ 编程语言

### Q6: 如何导出文档？

A: 点击 Toolbar 右侧的**导出按钮**（⤓），选择导出格式：
- **PDF** — 适合打印和分享
- **DOCX** — 适合在 Word 中继续编辑
- **HTML** — 适合网页发布

### Q7: 支持云同步吗？

A: MarkLite 目前不提供内置云同步。建议将 Markdown 文件夹放在 iCloud Drive、Dropbox、OneDrive 或 Google Drive 中自动同步。

### Q8: 可以自定义快捷键吗？

A: v0.5.0 支持快捷键自定义。打开 **设置面板**（Toolbar 右侧齿轮图标），切换到 **快捷键** 标签页，即可修改各项快捷键绑定。

### Q9: 是否有暗色模式？

A: 支持。点击 Toolbar 右侧的**主题切换按钮**（☀️/🌙）选择亮色、暗色或跟随系统设置。

### Q10: 文件树能否显示其他文件类型？

A: 目前文件树仅显示 `.md`、`.markdown` 和 `.txt` 文件以保持清洁。如需管理其他文件类型，建议使用系统文件管理器。

### Q11: 如何使用 Wiki-Link 导航？

A: 在 Markdown 中输入 `[[filename]]` 创建 wiki 链接。**单击**打开文档，**Ctrl+Click** 在新标签页打开。如果文档不存在，Ctrl+Click 会提示创建新文档。

### Q12: 片段管理器需要联网吗？

A: 不需要。片段保存到本地浏览器 localStorage，完全离线。但需注意 localStorage 有约 5MB 限制（根据浏览器而异）。

### Q13: 多光标编辑支持哪些操作？

A: 多光标支持所有基本编辑操作：
- 输入文本 — 所有光标位置同时输入
- 删除字符 — 所有光标同时删除
- 缩进/反缩进 — 所有行同时变化
- 剪切/复制/粘贴 — 操作影响所有光标

但不支持复杂的多光标模式切换（如矩形选择）。

### Q14: 如何导出 EPUB？

A: 点击 Toolbar 导出按钮，选择 **EPUB** 格式。EPUB 导出会自动提取文档 YAML Frontmatter 中的标题、作者等元数据。如不提供 Frontmatter，使用文件名作为标题。

### Q15: EPUB 导出支持哪些功能？

A: EPUB 支持：
- 丰富文本格式（加粗、斜体、代码等）
- 图片嵌入
- 目录生成
- 数学公式
- 但不支持 Mermaid 图表（会作为 PNG 嵌入）

### Q16: 预览区可以直接编辑吗？（v0.8.0+）

A: 可以！MarkLite v0.8.0 升级为 **Milkdown WYSIWYG 引擎**，分栏视图下预览区支持直接点击编辑：
- 点击任意段落、标题、列表项即可原地编辑
- 修改会自动同步到左侧 CodeMirror 编辑器
- 按 `Enter` 确认，`Esc` 取消

### Q17: AI Copilot 需要联网吗？（v0.8.0+）

A: 取决于您使用的模型。AI Copilot 支持：
- **云端模型**（OpenAI、Claude 等）— 需要联网和 API Key
- **本地模型**（Ollama 等 OpenAI 兼容接口）— 不需要联网，完全本地运行

### Q18: 如何使用幻灯片演示模式？（v0.7.0+）

A: 在 Markdown 文档中用 `---` 分隔每一页内容，然后按 `Ctrl+4` 或点击工具栏幻灯片图标进入演示模式。使用方向键翻页，按 `Esc` 退出。

### Q19: Git 面板支持哪些操作？（v0.7.0+）

A: Git 面板支持：查看当前分支与变更文件（staged/unstaged 区分）、查看 Diff 差异、输入 commit 信息提交、以及 Pull/Push 操作。需要系统已安装 Git，且当前文件夹为有效 Git 仓库。

### Q20: 如何安装插件？（v0.8.0+）

A: 点击左侧 ActivityBar 的插件图标打开插件面板，切换到"推荐"标签页，找到需要的插件点击安装即可。安装完成后插件立即激活。

如果熟悉 Vim，启用 **Vim 模式**可大幅提升编辑速度。按 `Esc` 进入 Normal 模式，使用 `hjkl` 导航，`dd` 删除行等标准 Vim 操作。

### 💡 Tip 2: 快速格式化

**Toolbar 按钮**比记住快捷键更直观。对于频繁使用的格式（加粗、列表等），使用 Toolbar 一键应用，无需担心快捷键。

### 💡 Tip 3: 通过右键菜单应用表格操作

不需要记住复杂的 Markdown 表格语法。在表格内**右键单击**，选择"添加行"或"删除列"等操作，更加直观。

### 💡 Tip 4: 使用打字机模式进行专注写作

在进行创意写作或需要深度专注时，启用**打字机模式**（`Ctrl+Shift+M`）隐藏所有 UI，行始终居中屏幕中央，沉浸感极强。

### 💡 Tip 5: 利用大纲进行长文档导航

对于超过 2000 字的长文档，频繁使用**大纲侧边栏**（右侧）快速跳转到不同章节，比 Ctrl+F 查找更快。

### 💡 Tip 6: 保存快照用于版本管理

在进行重大编辑前按 `Ctrl+S` 保存一个快照作为"检查点"。如果编辑出问题，可快速恢复到之前版本。

### 💡 Tip 7: Markdown + 脚注用于学术写作

使用 `[^1]` 脚注语法进行论文注释，MarkLite 会自动生成脚注链接。这比手工编号更清晰。

### 💡 Tip 8: 使用 Frontmatter 组织文档元数据

在文档顶部添加 YAML Frontmatter 块记录文档属性：

```yaml
---
title: 我的文章
author: 张三
date: 2026-04-10
tags: [写作, 技巧]
---
```

预览中会自动渲染为表格，便于查阅。

### 💡 Tip 9: 跨文件搜索进行内容索引

在打开文件夹的状态下，使用**跨文件搜索**（`Ctrl+Shift+F`）快速查找所有文件中的特定内容，适合管理项目笔记或知识库。

### 💡 Tip 10: 使用 Mermaid 图表增强可视化

在 Markdown 中插入代码块并使用 `mermaid` 语言标记，MarkLite 会自动渲染为图表：

```mermaid
graph LR
  A[开始] --> B[编辑]
  B --> C[预览]
  C --> D[导出]
```

### 💡 Tip 11: 命令面板快速访问

使用 `Ctrl+Shift+P` 打开命令面板，比在菜单中查找更快。对于不常用的功能（如导出 EPUB），直接搜索功能名称更高效。

### 💡 Tip 12: Wiki-Link 构建知识网络

对于长期维护的知识库或项目，使用 wiki 链接在相关笔记之间建立导航网络：
- 技术笔记中链接到相关概念
- 项目计划链接到设计文档和代码注释
- 学习笔记链接到参考资料和总结

### 💡 Tip 13: 创建通用片段模板

创建几个通用片段模板可显著提升工作流：
- `日志` — `### ${date} - ${cursor}`
- `代码` — ` ```language\n${cursor}\n``` `
- `引用` — `> ${cursor}\n>\n> ——出处`

### 💡 Tip 14: 多光标批量编辑列表

编辑列表时，使用 `Alt+D` 选中重复的项目名称，同时修改所有出现：
- `Alt+D` 选中所有相同的单词
- 输入新值，自动更新所有位置
- 比逐个手工修改快 10 倍

### 💡 Tip 15: 利用 YAML Frontmatter 管理文档属性

为所有导出的文档添加 Frontmatter 块，包含：
```yaml
---
title: 标题
author: 作者
date: 日期
tags: [tag1, tag2]
description: 简短描述
---
```

这样导出 EPUB 等格式时会自动应用这些属性。

### 💡 Tip 16: 用 WYSIWYG 预览快速格式化（v0.8.0+）

不想在原始 Markdown 中写格式标记？切换到**分栏视图**，直接在预览区编辑：
- 点击文本段落进入编辑模式
- 选中文字后使用弹出的 **InlineToolbar** 一键加粗、斜体或添加链接
- 比手写 `**text**` 更直观，适合非技术用户

### 💡 Tip 17: AI Copilot 辅助润色（v0.8.0+）

写完草稿后，利用 **AI Copilot** 快速改善文章质量：
1. 选中一段文字
2. 在 AI 面板点击"改写"或"总结"
3. 查看 AI 建议后点击"应用"插入到编辑器
4. 使用"翻译"功能快速生成双语版本

### 💡 Tip 18: 用幻灯片模式展示文档（v0.7.0+）

技术博客或会议分享？无需额外工具：用 `---` 分隔 Markdown，按 `Ctrl+4` 即变演示文稿。代码块、Mermaid 图表等在幻灯片中完整渲染。

### 💡 Tip 19: 思维导图辅助文章结构设计（v0.7.0+）

写作前先用思维导图（`Ctrl+5`）检视文档标题层级，确认结构合理再充实内容。也可在写作过程中随时切换思维导图，快速跳转章节。

### 💡 Tip 20: Git 面板集中管理文章版本（v0.7.0+）

将文章文件夹初始化为 Git 仓库，用内置 **Git 面板**保存每次重要修改的 commit，配合版本快照（§3.9）形成双层版本管理体系。
