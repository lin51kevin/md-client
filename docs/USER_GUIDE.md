## 1. 入门指南

### 1.1 安装与启动

**安装 MarkLite**：
- 从 [GitHub Releases](https://github.com/lin51kevin/md-client/releases) 下载对应平台的安装包
- macOS：双击 `.dmg` 文件，拖拽 MarkLite 到 Applications 文件夹
- Windows：运行 `.exe` 安装程序，按提示完成安装
- Linux：下载 `.AppImage` 文件，赋予执行权限：`chmod +x MarkLite-*.AppImage`

**从命令行启动**：
```bash
# macOS
open -a MarkLite /path/to/file.md

# Windows
MarkLite.exe C:\path\to\file.md

# Linux
./MarkLite-*.AppImage /path/to/file.md
```

### 1.2 第一次使用

1. **打开一个 Markdown 文件**或创建新文件（`Ctrl/Cmd + N`）
2. **熟悉三种视图**：
   - 📝 **编辑** — 仅显示编辑器
   - 📖 **分栏**（默认）— 编辑器与预览并排，实时同步
   - 👁️ **预览** — 仅显示渲染结果
3. **尝试编辑**：输入 Markdown，在预览区看到实时效果
4. **自动保存**：编辑 1 秒无变化后自动保存到磁盘

### 1.3 打开文件夹

MarkLite 支持**文件夹工作区**模式：

1. 点击 **Toolbar** 左上的**文件夹图标**，或按 `Ctrl+O`
2. 选择包含 Markdown 文件的文件夹
3. **文件树侧边栏**出现在左侧，显示所有 `.md` / `.markdown` / `.txt` 文件
4. 在文件树中**双击**任意文件快速打开
5. 使用**上级目录**按钮返回上级文件夹，或**刷新**按钮重载文件树

## 2. 编辑器基础

### 2.1 编辑操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 新建文件 | `Ctrl+N` | 创建无标题新文件 |
| 打开文件 | `Ctrl+O` | 打开单个 `.md` 文件 |
| 打开文件夹 | `Ctrl+Shift+O` | 打开文件夹（显示文件树） |
| 保存文件 | `Ctrl+S` | 手动保存（自动保存已启用） |
| 另存为 | `Ctrl+Shift+S` | 保存为新文件 |
| 关闭标签页 | `Ctrl+W` | 关闭当前编辑标签页 |
| 下一个标签页 | `Ctrl+Tab` | 导航到下一个标签页 |
| 上一个标签页 | `Ctrl+Shift+Tab` | 导航到上一个标签页 |

### 2.2 视图模式

**三种编辑视图** — 点击 Toolbar 右侧的视图按钮或使用快捷键切换：

| 模式 | 快捷键 | 说明 |
|------|--------|------|
| 📝 编辑 | `Ctrl+Alt+1` | 仅显示编辑器，最大化空间 |
| 📖 分栏 | `Ctrl+Alt+2` | 编辑器 + 预览并排，实时同步滚动 |
| 👁️ 预览 | `Ctrl+Alt+3` | 仅显示预览区，只读视图 |

### 2.3 焦点模式

在高度专注的环境下写作，隐藏 UI 干扰。Toolbar 右侧点击或快捷键激活：

| 模式 | 快捷键 | 说明 |
|------|--------|------|
| 📝 打字机模式 | `Ctrl+Shift+M` | 当前行始终居中，隐藏工具栏和状态栏 |
| 🎯 专注模式 | `Ctrl+Shift+F` | 仅显示编辑器，暗化周围 UI |
| 🔲 全屏模式 | `F11` 或 `Ctrl+Shift+P` | 占满整个屏幕（ESC 退出） |

### 2.4 主题切换

点击 Toolbar 右侧的**太阳/月亮图标**，或按 `Ctrl+Shift+T` 切换主题：

- ☀️ **亮色** — GitHub Light 风格
- 🌙 **暗色** — GitHub Dark 风格
- 🖥️ **跟随系统**（默认）— 自动跟随操作系统深色模式设置

## 3. 功能详解

### 3.1 编辑工具栏

**Toolbar** 提供一键格式化和快捷操作：

#### 文本格式（Text Formatting）
| 按钮 | 快捷键 | 效果 |
|------|--------|------|
| **B** 加粗 | `Ctrl+B` | 将选中文本包装为 `**bold**` |
| *I* 斜体 | `Ctrl+I` | 将选中文本包装为 `*italic*` |
| ~~S~~ 删除线 | `Ctrl+Shift+X` | 将选中文本包装为 `~~strikethrough~~` |
| `<>` 代码 | `Ctrl+E` | 将选中文本包装为 `` `code` `` |

#### 块级元素（Block Elements）
| 按钮 | 效果 |
|------|------|
| # 标题 | 在当前行或选中行前添加 `# `，向下依次为 H2、H3... |
| > 引用 | 在行前添加 `> ` 标记为引用块 |
| • 无序列表 | 在行前添加 `- ` 创建列表项 |
| 1. 有序列表 | 在行前添加 `1. ` 创建编号列表 |

#### 插入元素（Insertion）
| 按钮 | 效果 |
|------|------|
| 🔗 链接 | 弹出对话框，输入链接文本和 URL，插入 `[text](url)` |
| 🖼️ 图片 | 打开文件选择器，选择图片文件插入 `![alt](path)` 链接 |

### 3.2 右键上下文菜单

在编辑器内**右键单击**，出现上下文感知的菜单，包含 20+ 操作：

**通用操作**：
- ✂️ 剪切 / 📋 复制 / 📌 粘贴 / 🎯 全选
- 撤销 / 重做

**上下文操作**（根据光标位置自动显示相关操作）：

**在标题行**：
- ⬆️ 提升标题（H2 → H1）
- ⬇️ 降低标题（H1 → H2）
- 转换为其他标题级别

**在列表内**：
- ↪️ 缩进列表项
- ↩️ 反缩进列表项
- 转换列表类型（有序 ↔ 无序）

**在表格内**：
- ➕ 上方/下方插入行
- ➕ 左方/右方插入列
- ❌ 删除行/列
- ⬅️ 左对齐 / ⬆️ 中对齐 / ➡️ 右对齐

**在代码块/数学公式**：
- 📋 复制代码块内容

### 3.3 文件树侧边栏

打开文件夹后，左侧显示**文件树**，支持快速文件导航：

**文件树功能**：
- 📁 **文件夹展开/折叠** — 点击文件夹名左侧的 ▶️ 箭头
- 📄 **双击打开文件** — 在编辑器中打开选中的 `.md` / `.txt` 文件
- 🔄 **刷新** — 刷新按钮重载文件树（新增文件夹时使用）
- 📂 **选择其他文件夹** — 使用上级目录或选择按钮切换工作区
- 🔍 **文件筛选** — 仅显示 `.md`、`.markdown`、`.txt` 文件（自动过滤其他文件类型）

### 3.4 跨文件搜索

在打开文件夹的状态下，点击 Toolbar 中的**搜索图标**（🔍）或按 `Ctrl+Shift+F`：

1. **搜索面板**出现在右侧
2. 输入搜索关键词
3. 勾选 **Aa**（大小写敏感）或 **.*（正则表达式）**
4. 显示匹配结果（最多 200 条），结果格式为 `文件名 : 行号 : 匹配行内容`
5. **点击任意结果**快速跳转到该文件和行号

### 3.5 大纲侧边栏（TOC）

右侧显示当前文档的**目录大纲**，自动提取所有标题（H1-H6）：

**大纲功能**：
- 📍 **点击跳转** — 点击任意标题快速跳转到文档中该位置
- 🎚️ **折叠/展开** — 点击标题前的 ▶️ 箭头折叠其子标题
- 🔄 **实时更新** — 编辑时自动更新大纲结构

### 3.6 表格可视化编辑

在预览区**点击任意表格**，弹出编辑对话框：

**表格编辑功能**：
- ➕ **添加行** — 在表格上方或下方插入新行
- ➖ **删除行** — 删除指定行
- ➕ **添加列** — 在表格左方或右方插入新列
- ➖ **删除列** — 删除指定列
- ⬅️ / ⬆️ / ➡️ **对齐方式** — 改变列的文本对齐（左/中/右）
- ✏️ **编辑单元格** — 点击单元格直接编辑内容

对话框关闭后，修改内容自动更新到编辑器中的 Markdown 表格。

### 3.7 图片粘贴与拖拽

**粘贴图片**：
1. 将图片复制到剪贴板（或从文件管理器拖拽）
2. 在编辑器中光标定位到要插入图片的位置
3. **粘贴**（`Ctrl+V`）或**拖拽**图片文件到编辑器
4. 图片自动保存到 `assets/images/` 文件夹，生成 Markdown 格式 `![alt](path)`

**支持的图片格式**：PNG、JPG、JPEG、GIF、WebP

### 3.8 搜索与替换

按 `Ctrl+F` 打开**查找工具栏**：

- 输入搜索词，编辑器中自动高亮所有匹配项
- 勾选 **Aa** 开启大小写敏感搜索
- 勾选 **.*\*** 启用正则表达式搜索
- ↑ / ↓ 按钮导航到前一个/后一个匹配项

按 `Ctrl+H` 打开**替换工具栏**：

- 在"查找"字段输入搜索词，"替换"字段输入替换内容
- 🎯 **替换** — 替换当前匹配项
- 🔄 **全部** — 一次性替换所有匹配项

### 3.9 版本快照（本地历史）

**自动快照**：
- 每次按 `Ctrl+S` 手动保存时，自动创建当前文档的快照
- 最多保留最近 20 个快照

**访问快照**：
1. 点击状态栏右侧的**版本数字**（如 `5 snapshots`）
2. 弹出快照列表，显示创建时间戳
3. 点击任意快照预览其内容
4. 如要恢复，点击**恢复**按钮

### 3.10 写作统计

**实时统计面板**显示在状态栏右侧，包含：

| 指标 | 说明 |
|------|------|
| 字数（Words） | 文档中单词/中文字符总数 |
| 字符数（Characters） | 包含空格的总字符数 |
| 句子数（Sentences） | 按 `.!?;:` 分割的句子数 |
| 阅读时间（Reading Time） | 按 350 字/分钟计算的预计阅读时间 |

## 5. 常见问题

### Q1: 如何启用 Vim 模式？

A: 点击 Toolbar 左侧的**设置图标**，在弹出的**Settings**面板中勾选 **Vim Mode**，刷新编辑器即可。

### Q2: 图片粘贴后保存到哪里？

A: 图片自动保存到当前文件夹的 `assets/images/` 子目录。如果该目录不存在会自动创建。

### Q3: 文件分栏预览的滚动不同步？

A: 这是正常现象。MarkLite 会尽力同步编辑区和预览区的滚动位置，但由于渲染高度不同可能出现微小偏差。如需精确定位，可使用**大纲侧边栏**中的标题链接快速跳转。

### Q4: 能否使用 Jupyter Notebook 或其他格式？

A: MarkLite 目前仅支持 Markdown（`.md` / `.markdown`）和纯文本（`.txt`）格式。

### Q5: 支持哪些 Markdown 扩展？

A: MarkLite 支持以下扩展：
- **GFM**（GitHub Flavored Markdown）— 表格、删除线、任务列表等
- **KaTeX** — 数学公式（`$...$` 行内，`$$...$$` 块级）
- **Mermaid** — 流程图、时序图、状态图等
- **脚注** — `[^1]` 脚注定义和引用
- **Frontmatter** — YAML 元数据块
- **自定义容器** — `::: info` 等指令块
- **代码高亮** — highlight.js 支持 100+ 编程语言

### Q6: 如何导出文档？

A: 点击 Toolbar 右侧的**导出按钮**（⤓），选择导出格式：
- **PDF** — 适合打印和分享
- **DOCX** — 适合在 Word 中继续编辑
- **HTML** — 适合网页发布

### Q7: 支持云同步吗？

A: MarkLite 目前不提供内置云同步。建议将 Markdown 文件夹放在 iCloud Drive、Dropbox、OneDrive 或 Google Drive 中自动同步。

### Q8: 可以自定义快捷键吗？

A: v0.4.0 暂不支持快捷键自定义。此功能计划在 **v0.5.0** 实现。

### Q9: 是否有暗色模式？

A: 支持。点击 Toolbar 右侧的**主题切换按钮**（☀️/🌙）选择亮色、暗色或跟随系统设置。

### Q10: 文件树能否显示其他文件类型？

A: 目前文件树仅显示 `.md`、`.markdown` 和 `.txt` 文件以保持清洁。如需管理其他文件类型，建议使用系统文件管理器。

## 6. 提示与技巧

### 💡 Tip 1: 使用 VI 模式提升编辑效率

如果熟悉 Vim，启用 **Vim 模式**可大幅提升编辑速度。按 `Esc` 进入 Normal 模式，使用 `hjkl` 导航，`dd` 删除行等标准 Vim 操作。

### 💡 Tip 2: 快速格式化

**Toolbar 按钮**比记住快捷键更直观。对于频繁使用的格式（加粗、列表等），使用 Toolbar 一键应用，无需担心快捷键。

### 💡 Tip 3: 通过右键菜单应用表格操作

不需要记住复杂的 Markdown 表格语法。在表格内**右键单击**，选择"添加行"或"删除列"等操作，更加直观。

### 💡 Tip 4: 使用打字机模式进行专注写作

在进行创意写作或需要深度专注时，启用**打字机模式**（`Ctrl+Shift+M`）隐藏所有 UI，行始终居中屏幕中央，沉浸感极强。

### 💡 Tip 5: 利用大纲进行长文档导航

对于超过 2000 字的长文档，频繁使用**大纲侧边栏**（右侧）快速跳转到不同章节，比 Ctrl+F 查找更快。

### 💡 Tip 6: 保存快照用于版本管理

在进行重大编辑前按 `Ctrl+S` 保存一个快照作为"检查点"。如果编辑出问题，可快速恢复到之前版本。

### 💡 Tip 7: Markdown + 脚注用于学术写作

使用 `[^1]` 脚注语法进行论文注释，MarkLite 会自动生成脚注链接。这比手工编号更清晰。

### 💡 Tip 8: 使用 Frontmatter 组织文档元数据

在文档顶部添加 YAML Frontmatter 块记录文档属性：

```yaml
---
title: 我的文章
author: 张三
date: 2026-04-10
tags: [写作, 技巧]
---
```

预览中会自动渲染为表格，便于查阅。

### 💡 Tip 9: 跨文件搜索进行内容索引

在打开文件夹的状态下，使用**跨文件搜索**（`Ctrl+Shift+F`）快速查找所有文件中的特定内容，适合管理项目笔记或知识库。

### 💡 Tip 10: 使用 Mermaid 图表增强可视化

在 Markdown 中插入代码块并使用 `mermaid` 语言标记，MarkLite 会自动渲染为图表：

```mermaid
graph LR
  A[开始] --> B[编辑]
  B --> C[预览]
  C --> D[导出]
```
