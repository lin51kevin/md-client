## 三、优化方向（按优先级）

### 🔴 P0 — 阻断性问题（影响核心使用）

#### 1. 修复导出功能 Bug（进行中）
- PDF：InlineMath 不走预渲染、counter 顺序错误、`make_style` 签名不匹配、emoji 字体缺失
- DOCX：`make_style` 签名不匹配
- Frontmatter 未剥离、链接 URL 丢失、页码缺失

#### 2. Linux CJK 字体支持
- 当前 Linux 字体列表无 CJK 字体，PDF 中文全 tofu
- 待修复：已在 staged 代码中

### 🟠 P1 — 重要功能缺失（影响竞争力）

#### 3. TOC 导出支持
- Typora/Zettlr 均支持导出带目录的文档
- 当前仅侧边栏预览 TOC，无法导出到 PDF/DOCX
- **实现方案**：在 export_pdf.rs / export_docx.rs 中已有 TOC 生成逻辑（审核报告中显示已有代码），需验证是否正确工作

#### 4. LaTeX 公式渲染导出
- Typora 支持导出后 LaTeX 仍然渲染为高质量图片
- 当前 MarkLite 预览支持，导出只有源码或预渲染 PNG
- **方案**：前端 KaTeX → PNG 预渲染已实现（待修 bug），关键是确保 Rust 端正确嵌入

#### 5. 导出进度提示
- 大文档导出时用户无感知
- **方案**：在 `handleExport` 返回 `{ isExporting }` 状态，前端 UI 反馈

#### 6. 拼写检查
- 设置里开关是虚设的（只是把 browser spellcheck 属性传来传去）
- CodeMirror 6 无内置拼写检查，需要引入 `spell-checker` 或类似方案
- **参考**：`@codemirror/spell` 或原生 browser `spellcheck` 需配合 contenteditable

#### 7. PDF 页眉（文档标题）
- 当前只有页码"第 X 页"，无页眉
- 学术写作需要页眉显示文档标题

### 🟡 P2 — 体验提升（对标竞品）

#### 8. EPUB 导出
- Zettlr/Typora/Joplin 均支持
- 可用 ` epub-gen` 或 `md-to-epub` 库

#### 9. 幻灯片导出（Marp）
- Typora 支持一键导出幻灯片
- 可对接 Marp（`@marp-team/marp-cli`）

#### 10. HTML 离线包增强
- 当前 HTML 导出已内联 KaTeX CSS
- 建议内联 highlight.js 主题、Mermaid 渲染依赖
- 实现完全离线的单文件 HTML

#### 11. 深色模式导出
- 当前 PDF/HTML 导出始终浅色背景
- Typora/Zettlr 支持"以深色主题导出"

#### 12. 表格可视化编辑增强
- 当前表格编辑器功能基础（双击/右键打开）
- 建议：插入表格对话框（选择行列数）、表格工具栏（对齐、删除行/列）

#### 13. 图像处理
- 图片自动上传到图床（可选）
- 图片压缩
- 图片居中/对齐标记支持

### 🟢 P3 — 长期规划

#### 14. 云同步
- Obsidian（付费）/ Joplin / HackMD 均支持
- 可考虑：WebDAV、Dropbox、Google Drive 插件

#### 15. 协作编辑
- HackMD 是唯一原生支持的
- 实现成本高，可作为长期目标

#### 16. 插件系统
- Zettlr 有 Zettelkasten 插件
- Obsidian 插件生态丰富
- 可参考 CodeMirror 6 的 extension 机制

#### 17. 版本历史增强
- 当前基于 localStorage（仅浏览器端）
- 可增加基于 Git 的版本历史（对接工作目录）

#### 18. OCR 图片文字识别
- Joplin 支持
- 可接入 Tesseract.js

#### 19. 深色主题导出 PDF
- 当前固定白色背景
- 可导出时选择主题应用到 PDF

## 四、技术债务

| 问题 | 说明 |
|------|------|
| 测试覆盖率低 | 仅有 5 个测试文件，新增的 export-prerender 测试不完整 |
| Rust 前端代码重复 | export_pdf.rs 和 export_docx.rs 状态机约 300+ 行重复 |
| localStorage 版本历史 | 仅限单设备，无法跨设备同步快照 |
| 无 CI/CD 测试 | 仓库无自动化测试流程 |
| 权限配置简单 | fs 权限使用 `default`，未细分读写目录 |

## 五、总结

MarkLite 是一个功能完整的 Markdown 桌面编辑器，核心体验（分栏预览、多Tab、Vim、主题）已经做得相当扎实。与 Typora 相比，它有更好的开源性和可定制性。

**主要差距**：
1. **导出功能有 bug**（P0，修复中）
2. **LaTeX/TOC 导出**不如 Typora 完善
3. **无插件系统**，扩展性受限
4. **无云同步和协作**
5. **无幻灯片/EPUB 导出**

**可追赶的方向**：
- 短期：修 bug → 补齐导出功能（TOC/LaTeX/页码）
- 中期：EPUB/幻灯片、拼写检查、深色导出
- 长期：插件系统、云同步、协作

**对标 Typora 是合理的**，但 Typora 收费 $15，MarkLite 作为免费替代品在核心功能上已经覆盖了 80% 的场景。
