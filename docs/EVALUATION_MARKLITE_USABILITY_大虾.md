## 🔴 高优先级改进（阻塞性或高频影响）

### 1. 新建文档黑屏问题 [BUG]

**问题描述**: 工具栏和标签栏 + 按钮新建文档导致黑屏

**影响分析**:
- **用户流失率**: 首次使用即遇到黑屏，用户可能直接放弃
- **信任度**: 核心功能故障严重损害产品可靠性认知
- **使用频率**: 新建文档是最高频操作之一

**数据估算**:
```
假设日活用户: 100人
新建操作/用户/天: 5次
受影响操作: 工具栏(40%) + 标签栏(30%) = 70%
每日受影响次数: 100 × 5 × 0.7 = 350次
```

**修复建议**: 确保 Milkdown 初始化时内容不为空字符串

---

### 2. 全局文件搜索缺失 [功能缺口]

**问题描述**: 当前搜索仅支持已打开的文件，无法搜索整个工作目录

**竞品对比**:
| 工具 | 全局搜索 | 搜索结果预览 | 正则支持 |
|------|----------|--------------|----------|
| VS Code | ✅ | ✅ | ✅ |
| Obsidian | ✅ | ✅ | ✅ |
| Typora | ❌ | - | - |
| MarkLite | ❌ | - | - |

**用户场景**:
- "我记得写过关于 X 的内容，但忘了在哪个文件"
- "需要批量替换所有文件中的某个术语"
- "查找所有包含 TODO 的文件"

**建议实现**:
- 集成 `ripgrep` 或 Tauri 文件系统 API
- 支持 `.gitignore` 过滤
- 结果列表支持快速预览

---

### 3. 选中文字→AI 处理 [AI 集成增强]

**问题描述**: AI Copilot 只能手动触发，无法对选中内容直接操作

**用户场景分析**:
```
场景1: 选中一段代码 → 右键 → "解释这段代码"
场景2: 选中一段文字 → 右键 → "润色/翻译/总结"
场景3: 选中表格 → 右键 → "转换为图表"
```

**实现复杂度**: 🟡 中
- 需要扩展 EditorContextMenu
- 需要设计 prompt 模板系统
- 需要处理流式响应

**预期收益**:
- AI 功能使用率提升 3-5 倍
- 用户留存率提升（核心差异化功能）

---

### 4. 文件标签颜色标记 [组织效率]

**问题描述**: 多标签工作时难以快速识别文件类型或优先级

**用户行为研究**:
- 专业用户通常同时打开 5-15 个文件
- 视觉记忆比文字记忆快 60,000 倍
- 颜色编码可提升文件识别速度 40%

**建议方案**:
```typescript
// 右键标签 → 设置颜色
interface TabColor {
  id: string;
  name: string;
  color: string; // CSS color
}

const TAB_COLORS = [
  { id: 'red', name: '紧急', color: '#ef4444' },
  { id: 'yellow', name: '进行中', color: '#eab308' },
  { id: 'green', name: '已完成', color: '#22c55e' },
  { id: 'blue', name: '参考', color: '#3b82f6' },
  { id: 'purple', name: '个人', color: '#a855f7' },
];
```

## 🟡 中优先级改进（效率提升）

### 6. 快捷键可视化与自定义

**问题描述**: 用户不知道有哪些快捷键，也无法自定义

**当前状态**:
- 快捷键分散在 HelpModal 和 constants.ts
- 无自定义能力

**建议方案**:
```
设置面板 → 快捷键
├── 搜索快捷键
├── 按类别筛选 (文件/编辑/视图/AI)
├── 点击修改绑定
├── 冲突检测
└── 导出/导入配置
```

**参考实现**: VS Code 快捷键编辑器

---

### 7. 文件树 Hover 预览

**问题描述**: 不确定文件内容时不敢打开

**用户场景**:
- 文件树中有多个相似命名的文件
- 想快速确认文件内容再决定是否打开

**建议实现**:
- Hover 文件节点 500ms 后显示预览浮层
- 显示前 10 行内容
- 支持快速操作（在新标签页打开、复制路径）

---

### 8. 代码块折叠/展开

**问题描述**: 长文档中代码块占用过多视觉空间

**实现参考**:
```markdown
```javascript [collapsed]
// 长代码...
```
```

或基于 CodeMirror 的折叠功能

---

### 9. 导出面板统一

**问题描述**: 导出功能分散在菜单中， discoverability 差

**当前路径**:
- File → Export → PDF/DOCX/HTML/PNG
- 用户难以发现

**建议方案**:
- 工具栏添加统一导出按钮
- 点击后弹出导出选项面板
- 支持预览导出效果

---

### 10. Vim 模式状态显示

**问题描述**: 启用 Vim 模式后，不知道当前处于什么模式

**建议方案**:
- 底部状态栏显示 `NORMAL` / `INSERT` / `VISUAL`
- 模式切换时有视觉反馈
